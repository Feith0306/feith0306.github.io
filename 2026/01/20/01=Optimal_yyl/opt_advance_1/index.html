优化进阶系列 - 1. proximal 初步

1. 重新审视 Gradient Descent 算法：原问题的逐次逼近求解

考虑优化问题 \[ \min_x f(x) \]

针对优化目标函数 \(f(x)\)，如果将 \(\frac{1}{t}I\) 替代\(\nabla^2f(x)\)，则 \(f(\cdot)\) 在给定点 \(x\) 处的二阶近似可以表示为 \[ \hat{f}(z;x,t) = f(x) + \nabla f(x)^\top (z - x) + \frac{1}{2t} ||z - x||^2 \] 所以，当迭代至 \(x_k\) 时，可以通过优化 \(\hat{f}(\cdot;x_k,t)\) 来得到 \(x_{k+1}\)： \[ \begin{align} x_{k+1} =& \arg\min_{z} \hat{f}(z;x_k,t) \nonumber\\ =& \arg\min_{z} \left[f(x_k) + \nabla f(x_k)^\top (z - x_k) + \frac{1}{2t} ||z - x_k||^2\right] \end{align} \] 该优化问题的解析解为 \[ \begin{align} \label{eq:GD_f} x_{k+1} = x_k - t \nabla f(x_k) \end{align} \]

所以可以得出结论：

梯度下降算法是一种 逐次逼近 的方法

  • 逐次怎么逼近：
    • 先找目标函数 \(f\) 的上界 \(\hat{f}\)
    • 目标函数 \(f\) 虽然凸但是直接优化困难，取而代之去优化上界 \(\hat{f}\)
    • 上界 \(\hat{f}\) 具有良好的解析性质，可以直接得到最小值的显示表达式
    • 总结一句话：针对上界，找上界的最小值
  • 每次迭代都在求解近似目标函数 \(\hat{f}(z;x,t)\) 而不是原目标函数 \(f(z)\)
  • 在 \(x_k\) 处的近似目标函数 \(\hat{f}(z;x,t)\) 最小值点就是 \(x_{k+1}\)，并且 \(x_{k+1}\) 具有解析表达式 \(\eqref{eq:GD_f}\)

2. 不光滑目标函数

假设优化目标函数 \(f(x)\) 可以分解为两部分：\(f(x) = g(x) + h(x)\)，其中

  • \(g(x)\) 是一个光滑函数
  • \(h(x)\) 是一个不光滑函数
  • 两者均为凸函数

此时，由于 \(f(x)\) 有不光滑成分 \(h(x)\)，所以不能直接使用梯度下降算法来求解。这个时候又该怎么办呢？

  • 先不处理不光滑部分 \(h(x)\)，只针对光滑部分 \(g(x)\)，使用梯度下降算法来求解：
    • 在 \(x_k\) 处针对光滑部分 \(\hat{g}(z;x_k,t)\) 进行二阶近似: \[ \hat{g}(z;x_k,t) \approx g(x_k) + \nabla g(x_k)^\top (z - x_k) + \frac{1}{2t} ||z - x_k||^2 \]
    • 求解 \(\hat{f}(z;x_k,t) = \hat{g}(z;x_k,t) + h(z)\) 的最小值: \[ \begin{eqnarray} x_{k+1} &=& \arg\min_{z} \hat{g}(z;x_k,t) + h(z) \nonumber\\ &=& \arg\min_{z} \left[g(x_k) + \nabla g(x_k)^\top (z - x_k) + \frac{1}{2t} ||z - x_k||^2 + h(z)\right] \nonumber\\ &=& \arg\min_{z} \frac{1}{2t} ||z - \underbrace{[x_k - t \nabla g(x_k)]}_{w}||^2 + h(z) \label{eq:GD_gh} \end{eqnarray} \]

观察 \(\eqref{eq:GD_gh}\) 可知

  • 目标函数中的 \(\frac{1}{2t} ||z - \underbrace{[x_k - t \nabla g(x_k)]}_{w}||^2\)：
    • 希望 \(z\) 尽可能接近 \(w = x_k - t \nabla g(x_k)\)
    • 希望 \(z=x_{k+1}\) 尽可能按照这个规律求解：在 \(x_k\) 处沿着 \(\nabla g(x_k)\) 方向移动
  • 目标函数中的 \(h(z)\)：
    • \(z=x_{k+1}\) 的计算同时考虑到最小化 \(h(z)\)
    • 把 不好直接处理 的 \(h(z)\)（比如不可导、带约束、非光滑）附加一个 二次正则，变成一个 更容易解、且强凸 的子问题。

对不光滑部分 \(h(z)\)，又该如何继续处理呢？这就需要引入 proximal 算子。

3. proximal 算子的定义

定义 prox-1. (proximal 算子) 给定以下元素

  • 一个适当（proper）、下半连续（l.s.c.）的凸函数 \(h:\mathbb{R}^n\to(-\infty,+\infty]\)
  • 参数 \(t>0\)

则函数 h 的 proximal 算子 定义为 \[ \begin{eqnarray} \mathrm{prox}_{h,t}(w) = \arg\min_{z} \left[\underbrace{h(z) + \frac{1}{2t} ||z - w||^2}_{\hat{f}(z;w)}\right] \label{eq:prox_def} \end{eqnarray} \]

解读：

  • 关于 目标函数 \(\hat{f}(z;w) = h(z) + \frac{1}{2t} ||z - w||^2\) ：
    • 自变量是 \(z\)，而 \(w\) 是给定的参数
    • 最小值表达式与自变量 \(z\) 无关，只与参数 \(w\) 有关
  • proximal 算子的作用：
    • 将 参数 w （函数逼近的位置） 映射到该位置处近似目标 \(\hat{f}(z;w)\) 的最优解 \(z\)
    • \(\hat{f}(z;w)\) 的最优解 \(z\) 尽可能接近 参数 w，同时满足 h(z) 最小化
  • proximal 算子 \(\mathrm{prox}_{h,t}(w)\) 的参数 \(t\) ：
    • 当 \(t\) 越大时，映射越接近 参数 w；
    • 当 \(t\) 越小时，映射越接近 h(z) 最小化解
  • proximal 算子 \(\mathrm{prox}_{h,t}(w)\) 的光滑部分 \(\frac{1}{2t} ||z - w||^2\) ：
    • 给更新（\(w \to z\) / \(x_k\to x_{k+1}\)）加了“刹车”
    • 对于 病态问题（条件数大） / 非光滑点（ℓ1 的尖点） / 保持可行性（约束集） 尤其重要
  • prox 可以看成 次微分算子 ∂𝑓 的 resolvent
    • 这带来非常有用的稳定性与收敛分析工具（非扩张、平均算子、单调算子理论）
    • 这也解释了为什么很多“prox-化”的迭代更容易证明收敛、更不容易炸。
  • proximal 算子 \(\mathrm{prox}_{h,t}(w)\) 与 \(g\) 无关，仅与 \(h\) 有关

4. proximal gradient descent

回顾 \(\eqref{eq:GD_gh}\) \[ \begin{eqnarray} x_{k+1} &=& \arg\min_{z} \frac{1}{2t} ||z - \underbrace{[x_k - t \nabla g(x_k)]}_{w}||^2 + h(z) \nonumber\\ &=& \mathrm{prox}_{h,t}(x_k - t \nabla g(x_k)) \label{eq:GD_gh-1} \end{eqnarray} \] 上式进一步等价于 \[ \begin{eqnarray} x_{k+1} &=& x_{k} - t G_t(x_{k}) \label{eq:GD_gh-2} \end{eqnarray} \] 其中 \(G_t(x)\) 是 \(f=g+h\) 的 prox-化广义梯度算子，定义为 \[ \begin{eqnarray} G_t(x) &=& \frac{x-\mathrm{prox}_{h,t}(x-t\nabla g(x))}{t} \label{eq:GD_gh-3} \end{eqnarray} \]

5. proximal gradient descent 收敛性分析

如果

  • \(g\) 是 凸函数 且 \(\nabla g\) 是 L-Lipschitz 连续函数
  • \(h\) 是 凸函数 且 强凸函数
  • 固定步长 \(t\) 满足 \(0 < t < \frac{1}{L}\)

则

  • proximal gradient descent 收敛速度 \(\mathcal{O}(1/k)\)： \[ \begin{eqnarray} f(x_{k})-f^\star &\leq& \frac{1}{2tk} \|x_{0} - x^\star\|^2 \label{eq:PGD_convergence} \end{eqnarray} \]
