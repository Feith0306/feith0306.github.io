凸优化学习系列 - 6. 近端梯度下降法

目标：读完这篇博客，应当能

  • 理解近端梯度的必要性以及作用机制；
  • 学会如何用近端梯度法求解复合凸优化问题；
  • 知道近端梯度法的收敛速率及其与标准梯度下降的关系；
  • 掌握加速近端梯度法的原理与实现；

1. 问题描述

1.1 背景

对于凸优化问题： \[\begin{align} \label{eq:cvx_opt} \min_x f(x) \end{align}\]

  • 当 \(f\) 可微时，可采用 凸优化学习系列 - 4 中讨论的梯度下降法： \[\begin{align} x_{k+1} = x_k - t_k \nabla f(x_k). \end{align}\]
  • 当 \(f\) 不可微时，可采用 凸优化学习系列 - 5 中讨论的次梯度法： \[\begin{align} x_{k+1} = x_k - t_k p_k,\quad p_k\in \partial f(x_k). \end{align}\]

虽然次梯度法很通用（对目标函数可微性没有要求），但是收敛速度慢：典型复杂度为 \(O(1/\varepsilon^2)\)（或 \(O(1/\sqrt{k})\) ）。 此外，现实里大量问题不是完全不可微，而是“一部分光滑 + 一部分非光滑”。次梯度法将函数视为完全不可微，把可微部分的结构浪费掉了。因此我们想到，能否设计一种方法，既利用可微部分的梯度信息，又能处理非光滑部分，同时还能兼顾收敛性能？

1.2 目标函数的复合表达

根据上述讨论，我们将目标函数写成复合形式： \[\begin{align} f(x)=g(x)+h(x), \end{align}\] 其中，\(g\) 为可微部分，\(h\) 为不可微部分。 因此，原问题 \(\eqref{eq:cvx_opt}\) 重新表达为 \[\begin{align} \min_x [g(x)+h(x)]. \end{align}\]

2. 近端梯度下降（Proximal Gradient Descent, PGD）

2.1 光滑部分——二次上界

对于可微部分 \(g\)，基于 凸优化学习系列 - 4 中的讨论有： \[\begin{align} \min_x g(x) \quad \Rightarrow \quad \min_x \hat g(x;x_k) \end{align}\] 其中，\(\hat g(x;x_k) = g(x_k) + \nabla g(x_k)^\top (x - x_k) + \frac{1}{2t} \|x - x_k\|^2\) 是 \(g\) 在 \(x_k\) 处的二阶近似。

针对复合问题，我们可以在每次迭代时，对光滑部分 \(g\) 做二次近似，而对难处理的非光滑部分 \(h\) 暂时不操作。因此 \[\begin{align} \begin{split} x_{k+1} &= \arg\min_x \left[\hat g(x;x_k) + h(x)\right] \\ & = \arg\min_x \left[g(x_k) + \nabla g(x_k)^\top (x - x_k) + \frac{1}{2t} \|x - x_k\|^2 + h(x)\right] \\ & = \arg\min_x \left[\frac{1}{2t} \|x - (x_k - t \nabla g(x_k))\|^2 + h(x)\right]. \end{split} \end{align}\]

上式第二个等号到第三个等号的推导具体为： \[\begin{align} \begin{split} & \arg\min_x \left[g(x_k) + \nabla g(x_k)^\top (x - x_k) + \frac{1}{2t} \|x - x_k\|^2 \right] \\ =& \arg\min_x \left[\frac{1}{2t} \|x - x_k\|^2 + \nabla g(x_k)^\top (x - x_k) \right] \\ =& \arg\min_x \left[\frac{1}{2t} \|x + x_k\|^2 + \frac{1}{t} (x - x_k)^\top (t \nabla g(x_k)) \right] \\ =& \arg\min_x \left[\frac{1}{2t} \|( x - x_k ) + t \nabla g(x_k))\|^2 \right] - \frac{t}{2}(\nabla g(x_k))^2 \\ =& \arg\min_x \left[\frac{1}{2t} \|x - (x_k - t \nabla g(x_k))\|^2 \right]. \end{split} \end{align}\]

2.2 非光滑部分——近段映射

对不可微的凸函数 \(h\)，步长 \(t>0\)，定义近端映射 / 算子 为 \[\begin{align} \label{eq:prox_def} \mathrm{prox}_{h,t}(x)=\arg\min_z \frac{1}{2t}\|x-z\|_2^2 + h(z). \end{align}\] 它是在“\(z\) 离 \(x\) 不要太远”的前提下，让 \(h\) 尽量小，可理解为“对 \(h\) 做一次带二次正则的最小化”。

基于近端映射，式 \(\eqref{eq:prox_def}\) 中的更新可写成 \[\begin{align} \label{eq:pgd_update} x_{k+1} = \mathrm{prox}_{h,t}\big(x_k - t \nabla g(x_k)\big). \end{align}\]

  • 第一项逼迫 \(x\) 靠近“对 \(g\) 做梯度一步”得到的点 \(x-t\nabla g(x)\)；
  • 第二项逼迫 \(h(x)\) 小；
  • 近端映射只依赖于 \(h\)（以及步长 \(t\)），与 \(g\) 无关。
  • 当 \(h=0\) 时，\(x_{k+1} = x_k - t \nabla g(x_k)\)，退化为标准梯度下降。

问题一：为什么这个近端算子能给出 \(x_{k+1}\)？它的映射机制是什么？

问题二：prox 会不会比原问题更难求解？

解答：很多重要的 \(h\)，其 \(\mathrm{prox}_{h,t}\) 有闭式解或高效实现（需举例说明），且 prox 只依赖于 \(h\)，与 \(g\) 无关。

因此典型分工是：

  • \(g\)：可能很复杂，但只需要算梯度；
  • \(h\)：设计成“prox 好算”的正则/约束（\(\ell_1\)、指标函数、核范数等）。

问题三：如果 prox 不能精确算怎么办？

2.3. PGD 的“广义梯度”视角

如果定义如下映射（广义梯度）： \[\begin{align} G_t(x)=\frac{x-\mathrm{prox}_{h,t}(x-t\nabla g(x))}{t}. \end{align}\] 则迭代 \(\eqref{eq:pgd_update}\) 可写成 \[\begin{align} x_{k+1}=x_k - t_k\,G_{t_k}(x_k). \end{align}\]

这解释了“为什么近端梯度仍然是一阶法”：它每步只用到 \(\nabla g\)（部分一阶信息） 与一次 prox。

问题四：为什么把 \(G_t(x)\) 称为“广义梯度”？它的数学意义是什么？它和标准梯度/次梯度有什么关系？

3. PGD 的步长选择

由于 PGD 的步长体现在对光滑部分 \(g\) 的梯度下降中，因此步长选择与标准梯度下降类似（见凸优化学习系列 - 4）。

3.1 固定步长

为了使 \(g的\) 二次上界尽量可靠，取固定步长 \[\begin{align} t\le \frac{1}{L} \end{align}\] 其中 \(L\) 是 \(\nabla g\) 的 Lipschitz 常数，即 \[\begin{align} \|\nabla g(x)-\nabla g(y)\|_2 \le L \|x - y\|_2, \quad \forall x,y. \end{align}\] 如果 \(\nabla g\) 不是 Lipschitz 连续的，或者 \(L\) 未知，为了避免迭代发散，只能选择保守的更新步长，会导致更新效率降低。

3.2 回溯线搜索（Backtracking line search）

回溯准则：取 \(0<\beta<1\)，每次从 \(t=t_{\text{init}}\) 开始，如果 \[\begin{align} \label{eq:pgd_backtrack} g\big(x-tG_t(x)\big)\ >\ g(x)-t\nabla g(x)^\top G_t(x) + \frac{t}{2}\|G_t(x)\|_2^2, \end{align}\] 则缩小 \(t\leftarrow \beta t\)，直到满足为止，再做一次 prox 更新。

注意：这里判断的是 \(g\) 的充分下降/二次上界条件，而不是直接判断 \(f\)，这是复合结构下的关键差异。

问题五：标准梯度下降的回溯线搜索准则是 \[\begin{align} \label{eq:gd_backtrack} g(x - t \nabla g(x)) > g(x) - \alpha t \|\nabla g(x)\|_2^2, \end{align}\] 为什么 \(\eqref{eq:pgd_backtrack}\) 与 \(\eqref{eq:gd_backtrack}\) 不同？它们之间有什么联系？

4. PGD 的收敛性质

定理一：若 \(g\) 凸且 \(L\)-smooth，\(h\) 凸且 prox 可计算，则 PGD 更新 \(\eqref{eq:pgd_update}\) 在任意步长 \(t\le 1/L\) 下均满足 \[\begin{align} f(x^{(k)})-f^\star \le \frac{\|x^{(0)}-x^\star\|_2^2}{2tk}. \end{align}\] （回溯线搜索下：\(t = \frac{\beta}{L}\)）即收敛速率为 \(O(1/k)\)（或 \(O(1/\varepsilon)\)）。

问题六：根据 \(g\) 的不同性质，是否也有不同的收敛性质？

5. 加速近端梯度法（accelerated proximal gradient method）

5.1 动机

参考 博客1和博客2 中的 Nesterov 加速思想，对复合 \(g+h\) 也适用：只需每步 一次 prox，并只用最近两步信息。

常规梯度下降也可以用 Nesterov 加速，凸优化学习系列 - 4 中的内容需要补充。

5.2 加速近端梯度法的迭代

初始化 \(x_0 =x_{-1}\)。对 \(k=1,2,\dots\)： \[\begin{align} \label{eq:apgd_update} \begin{split} v & = x_{k-1}+\frac{k-2}{k+1}\big(x_{k-1}-x_{k-2}\big), \\ x_{k}& =\mathrm{prox}_{h,t_k}\big(v-t_k\nabla g(v)\big) \end{split} \end{align}\]

问题七：

  • 为什么要这样设计加速？
  • 这样为什么能加速？
  • 这种加速方法是否和普通梯度下降的 Nesterov 加速一致？
  • 变量 \(v\) 有什么数学意义？
  • 这样加速之后还是“下降”方法吗？

5.3 加速近端梯度法的回溯线搜索

简单策略：取 \(\beta<1\)，\(t_0=1\)。第 \(k\) 步从 \(t=t_{k-1}\) 开始，若 \[\begin{align} \label{eq:apgd_backtrack} g(x_k) > g(v)+\nabla^\top g(v)(x_k-v)+\frac{1}{2t}\|x_k-v\|_2^2, \end{align}\] 则缩小 \(t_k \leftarrow \beta t_k\)，并令 \[\begin{align} x_k =\mathrm{prox}_{h,t_k}(v-t_k\nabla g(v)). \end{align}\] 但它会“强迫步长递减”，更复杂的策略可以避免该问题。

问题八：为什么加速近端梯度法的回溯线搜索准则 \(\eqref{eq:apgd_backtrack}\) 与普通 PGD 的回溯线搜索准则 \(\eqref{eq:pgd_backtrack}\) 不同？它们之间有什么联系？

5.4 加速近端梯度法的收敛性质

定理二：若 \(g\) 凸且 \(L\)-smooth，\(h\) 凸且 prox 可计算，则加速近端梯度法 \(\eqref{eq:apgd_update}\) 在任意步长 \(t\le 1/L\) 下均满足 \[\begin{align} f(x^{(k)})-f^\star \le \frac{2L\|x^{(0)}-x^\star\|_2^2}{(k+1)^2}. \end{align}\] （回溯线搜索下：\(t = \frac{\beta}{L}\)）即收敛速率为 \(O(1/k^2)\)（或 \(O(1/\sqrt{\varepsilon})\)）。

这是一般一阶方法可达的最优收敛速度。

6. 实操：近端梯度下降求解凸优化的方法实现

6.1 步骤总结

  1. 拆分：写成 \(f=g+h\)
  • 把“可微的部分”放进 \(g\)；
  • 把“不可微的部分”放进 \(h\)。
  1. 确认：\(\nabla g\) 是否 Lipschitz（或能否回溯）
  • 若能估计 \(L\)，直接取 \(t\le 1/L\)；
  • 否则用回溯线搜索。
  1. 计算两件事
  • \(\nabla g(x)\)；
  • \(\mathrm{prox}_{h,t}(\cdot)\)（最好闭式/高效）。
  1. 选算法
  • 先用 PGD（稳定、常单调更好调试）；
  • 再尝试 APGD（更快但不单调、对“每步代价结构”更敏感）。
  1. 停止准则建议
    理论常用 \(f(x^{(k)})-f^\star\)，但 \(f^\star\) 不可得。可用：
  • 相对变化：\(\frac{|f(x^{(k)})-f(x^{(k-1)})|}{\max\{1,|f(x^{(k-1)})|\}} \le \tau\)；
  • 迭代差：\(\frac{\|x^{(k)}-x^{(k-1)}\|}{\max\{1,\|x^{(k-1)}\|\}} \le \tau\)；
  • 或广义梯度 \(\|G_t(x^{(k)})\|\) 小（若可算）。

6.2 伪代码总结

算法 1： 近端梯度下降（PGD）

\[ \begin{array}{l} \textbf{Algorithm 1: Proximal Gradient Descent}\\ \textbf{Input: } x^{(0)}\in\mathbb{R}^n;\ t>0\ (\text{or }t_0>0,\beta\in(0,1)\text{ for backtracking});\ K.\\ 1:\ \textbf{for } k=1,2,\ldots,K\ \textbf{do}\\ 2:\ \ \ \ \textbf{(optional backtracking)}\ \text{set } t\leftarrow t_0\ \text{(or reuse last }t\text{)}\\ 3:\ \ \ \ \textbf{repeat}\\ 4:\ \ \ \ \ \ \ y \leftarrow x^{(k-1)}-t\,\nabla g\!\left(x^{(k-1)}\right)\\ 5:\ \ \ \ \ \ \ x_{\text{trial}} \leftarrow \mathrm{prox}_{h,t}(y)\\ 6:\ \ \ \ \ \ \ \textbf{if } g(x_{\text{trial}})\le g(x^{(k-1)})+\nabla g(x^{(k-1)})^\top(x_{\text{trial}}-x^{(k-1)})+\frac{1}{2t}\|x_{\text{trial}}-x^{(k-1)}\|_2^2\\ 7:\ \ \ \ \ \ \ \textbf{then break}\\ 8:\ \ \ \ \ \ \ \textbf{else } t\leftarrow \beta t\\ 9:\ \ \ \ \textbf{until } \text{condition holds}\\ 10:\ \ \ \ x^{(k)} \leftarrow x_{\text{trial}}\\ 11:\ \ \ \ \textbf{(stopping)}\ \textbf{if }\frac{\|x^{(k)}-x^{(k-1)}\|_2}{\max\{1,\|x^{(k-1)}\|_2\}}\le \varepsilon\ \textbf{then break}\\ 12:\ \textbf{end for}\\ \textbf{Output: } x^{(k)}\ (\text{final iterate}),\ f(x^{(k)}).\\[2mm] \end{array} \]

算法 2：加速近端梯度下降（APGD）

\[ \begin{array}{l} \textbf{Algorithm 2: Accelerated Proximal Gradient Descent}\\ \textbf{Input: } x^{(0)}\in\mathbb{R}^n;\ x^{(-1)}=x^{(0)};\ t>0\ (\text{or }t_0>0,\beta\in(0,1)\text{ for backtracking});\ K.\\ 1:\ \textbf{for } k=1,2,\ldots,K\ \textbf{do}\\ 2:\ \ \ \ \alpha_k \leftarrow \frac{k-2}{k+1}\qquad(\text{momentum parameter})\\ 3:\ \ \ \ v^{(k-1)} \leftarrow x^{(k-1)}+\alpha_k\big(x^{(k-1)}-x^{(k-2)}\big)\qquad(\text{extrapolation})\\ 4:\ \ \ \ \textbf{(optional backtracking)}\ \text{set } t\leftarrow t_0\ \text{(or reuse last }t\text{)}\\ 5:\ \ \ \ \textbf{repeat}\\ 6:\ \ \ \ \ \ \ y \leftarrow v^{(k-1)}-t\,\nabla g\!\left(v^{(k-1)}\right)\\ 7:\ \ \ \ \ \ \ x_{\text{trial}} \leftarrow \mathrm{prox}_{h,t}(y)\\ 8:\ \ \ \ \ \ \ \textbf{if } g(x_{\text{trial}})\le g(v^{(k-1)})+\nabla g(v^{(k-1)})^\top(x_{\text{trial}}-v^{(k-1)})+\frac{1}{2t}\|x_{\text{trial}}-v^{(k-1)}\|_2^2\\ 9:\ \ \ \ \ \ \ \textbf{then break}\\ 10:\ \ \ \ \ \ \ \textbf{else } t\leftarrow \beta t\\ 11:\ \ \ \ \textbf{until } \text{condition holds}\\ 12:\ \ \ \ x^{(k)} \leftarrow x_{\text{trial}}\\ 13:\ \ \ \ \textbf{(stopping)}\ \textbf{if }\frac{\|x^{(k)}-x^{(k-1)}\|_2}{\max\{1,\|x^{(k-1)}\|_2\}}\le \varepsilon\ \textbf{then break}\\ 14:\ \textbf{end for}\\ \textbf{Output: } x^{(k)}\ (\text{final iterate}),\ f(x^{(k)}).\\[2mm] \end{array} \]
